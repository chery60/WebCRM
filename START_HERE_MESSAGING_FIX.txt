PROBLEM:
--------
Employees don't appear in "Start a direct message" dialog
Shows: "No other workspace members yet"

ROOT CAUSE:
-----------
Employees exist in 'employees' table but not in:
  ✗ 'users' table (needed for identity)
  ✗ 'workspace_memberships' table (needed for messaging)

SOLUTION:
---------
Run migration 027 to automatically sync employees to both tables

┌────────────────────────────────────────────────────────────────┐
│  STEP 1: Apply the Fix                                        │
└────────────────────────────────────────────────────────────────┘

Option A: Supabase Dashboard
  1. Open Supabase Dashboard → SQL Editor
  2. Copy content of: supabase/migrations/027_fix_employee_and_messaging.sql
  3. Paste and click "Run"

Option B: Supabase CLI
  $ supabase db push

┌────────────────────────────────────────────────────────────────┐
│  STEP 2: Verify & Test                                        │
└────────────────────────────────────────────────────────────────┘

1. Go to Messages → Click "New Message"
2. You should now see all employees!
3. Select one and send a test message

┌────────────────────────────────────────────────────────────────┐
│  WHAT THE FIX DOES                                            │
└────────────────────────────────────────────────────────────────┘

✅ Creates workspace_memberships for all employees
